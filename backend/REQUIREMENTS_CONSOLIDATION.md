# Backend Requirements Files - Consolidation Guide

## Current State

The backend has multiple requirements files serving different purposes:

### Main Requirements Files
1. **requirements.txt** - Auto-generated by pip-compile from requirements.in
2. **requirements.in** - Source file for pip-compile
3. **requirements-fixed.txt** - Manually edited with security updates

### Specialized Requirements Files
4. **requirements-test.txt** - Testing dependencies
5. **requirements-llm.txt** - LLM/AI model dependencies (heavy)
6. **requirements-config.txt** - Configuration management (watchdog, pyyaml)
7. **requirements-windows.txt** - Windows-compatible versions (no asyncpg)

## Key Differences

### requirements-fixed.txt vs requirements.txt

The `-fixed` version contains important security updates:

- `python-multipart==0.0.18` (fixes CVE-2024-53981)
- `python-jose[cryptography]==3.5.0` (fixes PYSEC-2024-232, PYSEC-2024-233)
- `cryptography==46.0.5` (fixes CVE-2024-12797)
- `aiohttp==3.13.3` (fixes multiple CVEs)
- `requests==2.32.4` (fixes CVE-2024-47081)

### requirements-llm.txt

Heavy dependencies for local LLM inference:
- `llama-cpp-python==0.2.90`
- `torch==2.5.1`
- `transformers==4.46.3`
- `sentence-transformers==3.3.1`

**Note**: These are OPTIONAL and only needed if running local LLM models. Most deployments use API-based LLMs (OpenAI, Anthropic) and don't need these.

### requirements-windows.txt

Windows-specific version that:
- Excludes `asyncpg` (requires C++ build tools)
- Uses `psycopg[binary]` for PostgreSQL
- Has older versions of some packages

### requirements-config.txt

Minimal dependencies for configuration:
- `watchdog>=3.0.0` - File system monitoring
- `pyyaml>=6.0` - YAML support

## Recommendations

### Option 1: Update requirements.in and Recompile (Recommended)

Update `requirements.in` with security fixes from `requirements-fixed.txt`, then:

```bash
pip-compile requirements.in
```

This ensures all dependencies are properly resolved.

### Option 2: Use requirements-fixed.txt as Main

Rename `requirements-fixed.txt` to `requirements.txt` and update documentation.

### Option 3: Consolidate into requirements.in

Merge all specialized requirements into `requirements.in` with optional extras:

```python
# requirements.in
fastapi[all]==0.129.0
# ... base dependencies ...

# Optional: LLM support
# llama-cpp-python==0.2.90  # Uncomment for local LLM
# torch==2.5.1
# transformers==4.46.3

# Optional: Configuration management
watchdog>=3.0.0
pyyaml>=6.0
```

## Installation Instructions

### Standard Installation (Linux/Mac)
```bash
pip install -r requirements.txt
pip install -r requirements-test.txt  # For testing
```

### Windows Installation
```bash
pip install -r requirements-windows.txt
pip install -r requirements-test.txt  # For testing
```

### With Local LLM Support
```bash
pip install -r requirements.txt
pip install -r requirements-llm.txt  # Heavy dependencies
```

### Development Installation
```bash
pip install -r requirements.txt
pip install -r requirements-test.txt
pip install -r requirements-config.txt  # For hot reloading
```
