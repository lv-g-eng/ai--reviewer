# 用户 GitHub 连接指南

## 如何连接你的 GitHub 账户

连接 GitHub 账户非常简单，只需几步即可完成。

### 步骤 1: 登录平台

1. 访问平台：http://localhost:3000（或你的平台地址）
2. 使用你的账户登录

### 步骤 2: 进入项目页面

1. 登录后，点击导航栏的 "Projects" 或 "项目"
2. 你会看到项目列表页面

### 步骤 3: 连接 GitHub

1. 点击 **"Connect GitHub"** 或 **"Add Project"** 按钮
2. 如果还未连接 GitHub，会提示你连接

### 步骤 4: 授权平台

1. 浏览器会跳转到 GitHub 授权页面
2. 页面会显示：
   - 应用名称：AI Code Review Platform
   - 请求的权限：
     - 访问你的仓库
     - 读取你的基本信息
3. 点击 **"Authorize"** 或**"授权"**按钮

### 步骤 5: 完成连接

1. 授权后，浏览器会自动跳转回平台
2. 你会看到 "GitHub Connected" 或 "GitHub 已连接" 的提示
3. 现在你可以看到你的 GitHub 仓库列表了

## 使用 GitHub 功能

连接 GitHub 后，你可以：

### 1. 查看仓库列表

- 在项目页面可以看到你的所有 GitHub 仓库
- 包括公开和私有仓库

### 2. 创建项目

1. 选择一个 GitHub 仓库
2. 点击 "Add Project" 或 "添加项目"
3. 平台会自动分析该仓库的代码

### 3. 自动代码审查

- 当你在 GitHub 上创建 Pull Request 时
- 平台会自动进行代码审查
- 审查结果会作为评论发布到 PR 中

### 4. 架构分析

- 平台会分析你的代码架构
- 检测潜在的架构问题
- 提供改进建议

## 常见问题

### Q1: 连接 GitHub 安全吗？

**是的，非常安全。**

- 使用 GitHub 官方的 OAuth 授权机制
- 平台只能访问你授权的内容
- 你可以随时撤销授权

### Q2: 平台能访问我的哪些数据？

平台只能访问：

- ✅ 你的 GitHub 用户名和基本信息
- ✅ 你的仓库列表
- ✅ 仓库的代码内容
- ✅ Pull Request 信息

平台**不能**：

- ❌ 修改你的 GitHub 账户设置
- ❌ 删除你的仓库
- ❌ 访问你的密码
- ❌ 代表你执行未授权的操作

### Q3: 如何断开 GitHub 连接？

有两种方式：

**方式 1: 在平台中断开**
1. 进入账户设置
2. 找到 "GitHub Connection" 或"GitHub 连接"
3. 点击 "Disconnect" 或 "断开连接"

**方式 2: 在 GitHub 中撤销**
1. 访问：https://github.com/settings/applications
2. 找到 "AI Code Review Platform"
3. 点击 "Revoke" 或 "撤销"

### Q4: 连接失败怎么办？

如果连接失败：

1. **刷新页面重试**
   - 有时网络问题会导致连接失败
   - 刷新页面后重新点击 "Connect GitHub"

2. **检查 GitHub 状态**
   - 访问：https://www.githubstatus.com/
   - 确认 GitHub 服务正常

3. **清除浏览器缓存**
   - 清除浏览器缓存和 Cookie
   - 重新登录平台

4. **联系支持**
   - 如果问题持续存在
   - 联系平台管理员或技术支持

### Q5: 可以连接多个 GitHub 账户吗？

目前每个平台账户只能连接一个 GitHub 账户。

如果需要切换 GitHub 账户：
1. 断开当前连接
2. 连接新的 GitHub 账户

### Q6: 连接后会自动分析所有仓库吗？

**不会。** 平台只会分析你主动添加的项目。

连接 GitHub 后：
- 你可以看到仓库列表
- 选择要分析的仓库
- 只有选中的仓库会被分析

### Q7: 平台会修改我的代码吗？

**不会。** 平台只读取代码进行分析，不会修改你的代码。

平台只会：
- 读取代码内容
- 分析代码质量和架构
- 在 PR 中添加审查评论

### Q8: 私有仓库也能分析吗？

**可以。** 如果你授权了访问私有仓库的权限，平台可以分析私有仓库。

你的私有代码：
- 只有你能看到分析结果
- 不会被分享给其他用户
- 受到严格的安全保护

## 授权权限说明

当你授权平台访问 GitHub 时，平台请求以下权限：

### repo（仓库访问）

**包括：**
- 读取仓库代码
- 读取 Pull Request
- 在 PR 中添加评论
- 读取仓库元数据

**不包括：**
- 删除仓库
- 修改仓库设置
- 强制推送代码

### read:user（读取用户信息）

**包括：**
- 读取用户名
- 读取邮箱地址
- 读取头像

**不包括：**
- 修改用户信息
- 访问私密信息

## 隐私和安全

### 数据保护

- 🔒 所有数据传输使用 HTTPS 加密
- 🔒 GitHub 访问令牌安全存储
- 🔒 代码分析在安全环境中进行
- 🔒 分析结果只有你能看到

### 数据使用

平台使用你的 GitHub 数据仅用于：
- 代码质量分析
- 架构分析
- 生成审查报告

**不会用于：**
- 训练 AI 模型（除非你明确同意）
- 分享给第三方
- 商业用途（除了提供服务）

### 数据保留

- 你的 GitHub 访问令牌会一直保存，直到你断开连接
- 分析结果会根据数据保留政策保存
- 你可以随时删除项目和分析结果

## 故障排查

### 问题 1: 点击"Connect GitHub"没有反应

**可能原因：**
- 浏览器阻止了弹出窗口
- JavaScript 被禁用

**解决方案：**
1. 允许浏览器弹出窗口
2. 确保 JavaScript 已启用
3. 刷新页面重试

### 问题 2: 授权后跳转到错误页面

**可能原因：**
- 网络连接问题
- 平台配置问题

**解决方案：**
1. 返回平台首页
2. 重新尝试连接
3. 如果问题持续，联系支持

### 问题 3: 看不到仓库列表

**可能原因：**
- GitHub 连接未完成
- 权限不足

**解决方案：**
1. 检查 GitHub 连接状态
2. 重新授权平台
3. 确认已授予仓库访问权限

### 问题 4: 无法添加私有仓库

**可能原因：**
- 未授权访问私有仓库

**解决方案：**
1. 断开 GitHub 连接
2. 重新连接
3. 在授权页面确保选择了私有仓库访问权限
